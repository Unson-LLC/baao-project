## 2. 成果物の仕様

### 形式
- **PPTX ではなく、1スライド = 1枚の画像ファイル**（PNG または JPG）
- 編集可能ファイル（Figma / Illustrator など）は不要、画像のみで納品

### 解像度・サイズ
- **16:9 アスペクト比**
- **推奨: 1920 × 1080 px**（フルHD）
- Zoom画面共有・大画面投影で文字が潰れない解像度

### ファイル名
- 連番形式: `slide-01.png`, `slide-02.png`, ... `slide-15.png`
- 15枚すべて同じファイル形式で揃える

### 言語
- 日本語（漢字・ひらがな・カタカナ・必要に応じて英語）

### 3-2. Claude Code とは何か

外注者の方は知らないかもしれないので説明します:

**Claude Code** = Anthropic 社が提供する AI ツール。ChatGPT のように会話するのに加え、**自分のPC内のファイルを読み書きしたり、作業を実行してくれる**ところが特徴。「会話する AI」ではなく「**業務を任せる AI**」というポジション。

Desktop アプリで動く（ターミナル不要）。

### 3-3. 想定参加者（ペルソナ）

**A: 40代の管理職タイプ**
- 中堅企業の課長〜部長
- チームに部下が10人前後いる
- ChatGPT は週数回使う程度の経験
- Claude Code はまだ触っていない
- 部下から「AI で業務効率化したい」と相談を受けて、自分も学びたい
- 経費精算で月数千円〜数万円なら通せる立場
- 「失敗したくない」「部下にちゃんと教えられるようになりたい」が動機

### 3-4. セミナーの核心メッセージ

このセミナーで伝えたい**たった1つのこと**:

> **「Claude Code は『使い方』ではなく『何を渡すか』で価値が決まる」**

ツールの操作方法ではなく、**AIに何をどう頼むかという発想**を持って帰ってもらうことが目的。

### 3-5. 主要コンセプト3つ

1. **Claude Code は「委任する道具」**: ChatGPT が会話、Copilot が補完、Claude Code は委任
2. **「仕様渡し」より「物語渡し」**: 「グラフを作って」より「経営会議で3秒で状況がわかる資料が欲しい」と渡す
3. **任せる/任せない の線引き**: 全部 AI に任せるのではなく、自分が判断する領域を残す

### 3-6. BAAO 実践AI道場のブランドトーン

- **道場・修行の世界観**（道場、修行者、書院生、師範会、立合い、秘伝書 などの言葉を使う）
- 真面目で実務的、でも堅すぎない
- 経営者・管理職向け（学生向けではない）
- 「最新トレンド追跡」より「本質を実装する」スタンス

### スライド 03: 3つの道具の対比表

**ねらい**: ChatGPT / Copilot / Claude Code の違いを3秒で理解させる。

**載せる要素**: 3列の比較表

| | ChatGPT | Copilot | Claude Code |
|---|---------|---------|-------------|
| 役割 | 会話する道具 | 補完する道具 | **委任する道具** |
| 比喩 | 先生 | 助手 | **部下** |
| アウトプット | チャット返信 | コード補完 | **ファイル更新+作業実行** |

→ Claude Code の列だけ強調デザイン

### スライド 04: 部下に説明するなら

**ねらい**: 管理職が部下に Claude Code を説明する一言を提供する。

**載せる要素**:

> **「AI 部下が PC の中で実際に作業してくれるツール」**

### スライド 05: Desktop アプリ3手順インストール

**ねらい**: 「3クリックで入る」を視覚化する。

**載せる要素**: 3ステップ
1. claude.com/code を開く
2. インストーラーをダウンロード
3. ダブルクリック → 起動 → Claude.ai でサインイン

---

### スライド 06: 詰まりがちな3点

**ねらい**: 入り口でのつまずきを先回りで安心させる。

**載せる要素**: 3点の対処
1. **起動後にセキュリティ警告**: Mac は「プライバシーとセキュリティ」で許可、Windows は SmartScreen で「詳細情報」→「実行」
2. **フォルダ追加してもファイルが見えない**: 権限を確認、別フォルダで試す
3. **何を書けばいいか分からない**: 「このフォルダで何ができる？」が万能の最初の一言

---

### スライド 07: 仕様渡し vs 物語渡し

**ねらい**: セミナーの核心メッセージを視覚化。最重要スライドの1つ。

**載せる要素**: 左右の対比

| 仕様渡し（×） | 物語渡し（◎） |
|--------------|----------------|
| 「議事録を要約して」 | 「部下5人の議事録から、月曜の部門会議用に①顧客要望トップ3 ②次のアクション ③ブロッカー を表形式で」 |

矢印で「仕様 → 物語」の変換を強調。

---

### スライド 08: 物語渡しの「普遍フレーム」

**ねらい**: 全ての業務（資料・メール・分析・なんでも）に使える普遍的な3要素。デモ②（経営会議スライド）の直後に、フレーム化して理解させる。

**使うタイミング**: デモ②の終わり、応用範囲を広げる位置づけ。

**載せる要素**: タイトル「物語に含めるべき3要素」+ 3項目

- **① 誰向けに作るのか**: 経営層 / 顧客 / 部下 / etc.
- **② どんな場面で使うのか**: 何分で読まれるか、対面 or 非対面
- **③ 何を伝えたいのか**: 核心メッセージ1行

下部に1行添える: 「**この3要素を Claude Code に渡すだけで、出力の精度が劇的に変わる**」

---

### スライド 09: メール文面に「物語」を渡すなら

**ねらい**: P8 の普遍フレームを、**メール文面という具体ケース**に当てはめた応用版。デモ③（3トーン使い分け）の直前で、メール特有の要素に絞って提示する。

**使うタイミング**: デモ③に入る直前。「メールで物語を渡すって、具体的にこういうこと」と理解を1段深める。

**載せる要素**: タイトル「メール文面に物語を渡すなら」+ 3項目

- **① 相手は誰か**: 関係性（浅い/深い）/ 立場（上司/部下/取引先）/ 性格（論理派/感情派）
- **② どう読まれるか**: 月曜朝の冒頭 / 移動中スマホ / 集中時間に PC
- **③ どう動いてほしいか**: 即返信 / 期日内に対応 / 関係性維持だけでOK

下部に1行添える: 「**メールは『内容』より『誰が・どう読み・どう動くか』を渡すと、トーンが一発で決まる**」

---

### P8 と P9 の書き分け方針

| | P8 普遍フレーム | P9 メール特化応用 |
|---|---------------|------------------|
| 抽象度 | 抽象（あらゆる業務に使える） | 具体（メール文面のみ） |
| 役割 | デモ②の振り返り + 普遍化 | デモ③の予告 + 専門化 |
| 視点 | アウトプット設計の3要素 | メール特有の3要素（読まれ方・期待行動が中心） |

→ **同じ「物語渡し」を抽象→具体の流れで2段階で見せる構造**

## 5. デザイントーン指示

### 雰囲気
- **シンプル・上品・読みやすい**
- ビジネスシーン（経営者・管理職向け）に堪える格
- 道場の世界観を匂わせる（強すぎず）

### 色
- **メインカラー: 1〜2色に絞る**（外注者の判断で選定OK、ただし以下のNGに注意）
- 強調色は1色のみ（「物語渡し」を強調する箇所など）

### フォント
- 日本語: 明朝系 or サンセリフ系（ゴシック）、外注者判断
- 英語: サンセリフ
- 本文サイズ: Zoom画面共有でも読める大きさ（小さすぎないこと）

### レイアウト
- 1スライド1メッセージが原則
- 詰め込みすぎず、余白を活かす
- 表は3列までを推奨
