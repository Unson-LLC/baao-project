## ネタ2: それでも残る「価値を決める力」

### なぜ「価値を決めること」だけは人間にしかできないのか

AIは「答えを出す」ことはできる。でも「**何が価値あるか**」を決めることはできない。なぜなら:

1. **価値は主観**: 「この絵が美しい」「この料理が美味い」「この経験が有意義」— これらは事実ではなく**人間の感覚**に根ざす
2. **価値は関係性の中で生まれる**: ブランド、信用、信頼 — 「誰が言ったか」で価値が変わる
3. **価値は文脈依存**: 同じものでも、文化、時代、コミュニティで価値が180度変わる
4. **価値は「決める」行為**: 分析じゃない。選び取ること。AIは分析できるが「これだ」と**決められない**

> **キーフレーズ**: AIは「何が正しいか」を答えられる。でも「何が価値あるか」は人間が決める。

### 具体例で考える

- **Art**: AIが描いた絵と人間が描いた絵。物理的には同じでも、「誰が描いたか」で人間は価値を変える
- **Music**: AI作曲と人間作曲。ファンが感じる「共鳴」は人間相手にしか起きない
- **Business**: AIが作った提案書と人間が作った提案書。「この人が裏付けてる」という信用が価値
- **Teaching**: AIチューターと人間の師匠。「この人についていきたい」という感情が学びを深める

## ネタ3: 15-20年先 — 「人間らしさ」が全ての価値になる

### 人間の勝ち筋: 人間関係・ブランド・信用

AIが「できること」を全部代替しても、以下は**人間と人間の間**でしか生まれない:

| 領域 | なぜ人間にしかできないか | 具体イメージ |
|---|---|---|
| **ブランド** | 「誰が作ったか」に価値が宿る。AIブランドには感情移入しない | パティシエの名前入りのケーキ、職人のサイン入り作品 |
| **信用・信頼** | 信頼は「この人なら大丈夫」という**人間の感覚**。データでは構築できない | 紹介で決まる仕事、顔を知っているから任せられる案件 |
| **コミュニティ** | 「同じ場にいる」という共感。AIコミュニティには帰属意識が生まれない | 道場、部活、秘密基地。BAAOのような場の価値 |
| **体験** | 「あの人に会った」「あの場にいた」という**一期一会の経験** | ライブ配信、ライブ演奏、飲み会、旅 |
| **師弟関係** | 「この人についていきたい」という**感情**。指示ではない | BAAOの師範と修行者の関係 |

> **キーフレーズ**: 15年後、「何ができるか」で評価される世界は終わってる。「誰がやったか」「誰とやったか」で評価される。

### これは歴史の転換点

- 工業化時代: 「誰が作ったか」より「安く均質に作れるか」が価値 → 人間は機械の一部
- 情報化時代: 「情報を持ってるか」が価値 → 人間は情報の中継
- **AI時代**: 「誰が価値を決めたか」が価値 → **人間は人間に戻る**

## ネタ4: 直近5年 — 「一人GTMエンジニア」が最強の職業

### GTMエンジニアとは何か

一言で: **一人でプロダクトを市場検証できる人**

- **GTM = Go-To-Market**: プロダクトを市場に持っていく全工程
- 従来は「エンジニア + PM + マーケ + 営業」のチームでやっていたこと
- AI時代: **一人で全部できる**。しかも高速で回せる

### なぜ強いか

1. **プロダクトは誰でも作れる時代**: Cursor、Devin、v0、Bolt等で非エンジニアでも作れる
2. **差別化は「市場検証の速度」**: 作って終わりじゃない。市場に投げて反応見て改善する速度が全て
3. **一人で回せる**: エンジニアリング + マーケ + 営業 + ヒアリング + PM — この全部をAIで10倍速に
4. **組織のオーバーヘッドがない**: 会議ゼロ、稟議ゼロ、決裁即実行

### GTMエンジニアが持つべき5技能

| 技能 | AIでどう補うか |
|---|---|
| プロダクト開発 | Cursor/Claude Code/v0で一人でフルスタック開発 |
| マーケティング | AIでLP作成、広告文案、SEO分析を自動化 |
| 営業・顧客対話 | AIでリード獲得→ヒアリング→提案まで自動化 |
| データ分析 | AIで即座にABテスト分析、ユーザー行動分析 |
| プロダクトマネジメント | AIでタスク管理、ロードマップ、優先順位付け |

> **キーフレーズ**: 「プロダクトを作る」はAIに任せていい。大事なのは「何を検証するか決めて、市場に投げて、反応を読んで、次を決める」こと。これが価値を決める力の実践。

## ネタ5: 「価値を決める力」を鍛える3つの習慣

### 習慣1: 「これが価値だ」と言い切る練習
- AIは「AもBも考えられます」と言う。人間は「これだ」と言い切る
- 毎日1つ、「今日これが一番良かった」を人に言う
- **なぜ重要**: 価値を決める筋肉は使わないと衰える

### 習慣2: 人との対話を増やす
- 価値は人間の間でしか生まれる → 人と会う時間を増やす
- オンラインじゃない場。顔を合わせる。飲みに行く。
- **なぜ重要**: 人間関係・信用・ブランドは「会った数」で積み上がる

### 習慣3: 早く作って早く捨てる
- GTMエンジニア型の実践。アイデアが出たら即プロトタイプ → 即市場検証
- 「完璧にしてから出す」はAI時代の悪手。「粗くても早く出す」が正解
- **なぜ重要**: 市場の反応 = 「世の中が価値をどう決めるか」のリアルタイム学習

## 想定Q&Aネタ

**Q: プログラマーはAIに代替されますか？**
→ 「プログラミング作業」は代替される。でも「何を作るか決めて、AI出力を評価・統合できるエンジニア」の価値は上がる。さらに言えば、一人でマーケも営業もPMもやれるGTM型エンジニアが最強。

**Q: AI学ぶなら何から？**
→ まず使う。学ぶより触る。毎日10分でいい。そのうち「これAIに任せられるな」と感覚が育つ。

**Q: 年齢は関係ありますか？**
→ 関係ない。経験（ドメイン知識）があるほど「価値を決める力」が強い。50代の営業マンの「顧客の本音がわかる」はAIには不可能。経験 x AI = 最強コンボ。

**Q: 子供の教育はどうすべき？**
→ 暗記より「問う力」「価値を決める力」を鍛える。「これが好き」「これは価値ある」と自分で言える子が強い。AIが答える時代に、問いを作れて価値を決められる人が一番強い。

**Q: ブランドや信用って具体的にどう身につけるの？**
→ 小さくていい。「この人に任せれば大丈夫」と言われる人になる。約束を守る、成果を出す、人を紹介する。地味だけど、15年後これが全て。

**Q: GTMエンジニアってどうなればなれるの？**
→ 特別な資格はない。AI使って何か作って、市場に出して、反応見て、改善する。このサイクルを回し続ける人。最初は小さなツールでもOK。
