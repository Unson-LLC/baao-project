## ペルソナA：聴衆の脳

**A の関心ピラミッド**:
1. **自分の業務にどう使うか** — 議事録、報告書、メール、会議資料
2. **部下にどう教えるか** — 社内展開、レバレッジ
3. **課金リスク** — 経費通すときに説明できないと困る
4. **情報漏洩・コンプラ** — 社内データを入れていいか不安

**A が嫌うもの**:
- 講師の自己紹介に5分使う
- 専門用語連発
- 哲学的な話
- 「俺はこんなにすごい」自慢

**A が好むもの**:
- 「部下が今やってる業務」が具体例で出てくる
- 「社内に1人いると変わる」レバレッジ感
- リスク管理の現実論
- 決裁者目線の料金解説

## セクション2: Claude Code とは何か（5分・0:03 - 0:08）

### トーク

```
[0:03-0:04 結論先出し]
「Claude Code を一言で言うと、AIに業務を委任する道具です。
 ChatGPTやCopilotとはここが違います。」

[0:04-0:06 3つの道具の対比]
「皆さんがよく使う AI ツールと比較します。

 ChatGPT は『会話する道具』。聞けば答えてくれる先生。
 Copilot は『補完する道具』。書いてる横で提案してくれる助手。
 Claude Code は『委任する道具』。
   『これやっといて、終わったら報告して』が成立する部下。

 違いの本質は、Claude Code は自分のPC環境で動くこと。
 ファイルを直接読んで、書いて、コマンドを実行する。
 だから『議事録ファイル全部に目を通して、要点を Notion に書き込んで、
 終わったら Slack で教えて』ができる。」

[0:06-0:07 部下にどう説明するか]
「部下に説明するなら、こう言います。
 『AI 部下が PC の中で実際に作業してくれるツール』。
 ChatGPT は会話だけで完結、
 Claude Code は会話の結果がファイルに残る。
 ここが大きな違い。」

[0:07-0:08 ハマる人 vs ハマらない人]
「最後に、ハマる人とハマらない人の見分け方。
 ハマる人: 業務を分解して人に振った経験がある人
 ハマらない人: 全部自分でやらないと気が済まない人

 管理職の皆さんは多分、前者です。
 部下に仕事を振る感覚が、そのまま Claude Code に使えます。」
```

### スライド

- **スライド3**: 3つの道具の対比表
  | | ChatGPT | Copilot | Claude Code |
  |---|---------|---------|-------------|
  | 役割 | 会話する道具 | 補完する道具 | **委任する道具** |
  | 比喩 | 先生 | 助手 | **部下** |
  | アウトプット | チャット返信 | コード補完 | **ファイル更新+作業実行** |

- **スライド4**: 「部下に説明するなら」一言サマリ

### 心理設計

- A の脳「部下に説明できるか」を直接解消
- 「業務を振る感覚」と結びつけて A の経験値を活用
- 「ハマる人=管理職」と暗示して自信を持たせる

## セクション5: 課金・情報漏洩・社内展開のリアル（7分・0:37 - 0:44）

### トーク

```
[0:37-0:39 課金の現実]
「まず一番気になる課金の話。
 個人プラン: 月20ドル〜（個人で経費精算しやすい範囲）
 プロプラン: 月100ドル〜（がっつり使う人向け）
 チームプラン: ユーザー単位、社内展開時に検討

 使いすぎ防止の3つのコツ:
 ① モデル選択で安いものに切り替え
 ② 1日の上限を設定
 ③ 大きいタスクは事前にコスト見積もり

 経費通すとき、『ChatGPT は会話用、Claude Code は作業実行用』
 という説明が一番通りやすいです。」

[0:39-0:41 情報漏洩のリスク管理]
「次、情報漏洩。これは重要です。
 Claude Code は自分のPCで動きますが、
 中身は Anthropic のサーバーに送信されてます。
 
 だから:
 ① 顧客の個人情報・契約書類は入れない
 ② 社外秘の経営数字は加工してから渡す
 ③ パスワード・認証情報は絶対入れない

 逆に、入れていいもの:
 ・社内の議事録（人名は伏字に）
 ・公開情報の分析
 ・自分の業務日報

 心配な方は『社内で正式に AI 利用ポリシーを確認してから』が
 ベストです。」

[0:41-0:44 社内展開で詰まる典型パターン3つ]
「最後、社内展開の話。
 100社見てきた人の話で、AI導入で失敗する典型3パターン:

 ① ツール名から導入
   『Claude Code を導入する』が目的化 → 何のためか分からなくなる
   正解: 『どの業務の何時間を減らすか』から始める

 ② 一部署で成功 → 全社展開で失敗
   一部署で成功した理由は『その人たちが優秀』だっただけ
   全社展開すると、別の部署では文脈が違って動かない
   正解: 全社展開ではなく、部署ごとに独立した導入を

 ③ 評価制度を先に変える
   『AI使えてる人を評価する』を先にやると、混乱だけが残る
   正解: 業務設計→ツール→評価、の順

 皆さんが社内展開の意思決定する立場なら、
 この3つだけは覚えて帰ってください。」
```

### スライド

- **スライド10**: 課金プラン3階建て + 使いすぎ防止3点
- **スライド11**: 情報漏洩 入れていい/ダメ 一覧
- **スライド12**: 社内展開 失敗3パターン

### 心理設計

- A の不安「課金で説明できない」「情報漏洩怖い」を**具体的に解消**
- 「100社見てきた人の話」で**外部の権威性**を借りる（自慢にならない）
- 失敗パターンを先に出すことで、A の「失敗したくない」欲求を直撃
