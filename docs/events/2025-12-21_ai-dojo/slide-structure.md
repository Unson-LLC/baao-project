## スライド8: なぜ効くのか - 立ち位置の力

**人間同士の会話と同じ**

- 初対面の人 → 全て説明が必要
- 長年の同僚 → **あなたの立ち位置を知っているから**話が早い

**AIも同じ**
- コンテキストなし → 毎回ゼロから説明
- コンテキストあり → **あなたの立ち位置を理解した**パートナー

→ **立ち位置 = 仕事におけるあなたの「座標」**

→ **AIの賢さ × 立ち位置整理 = 真の生産性向上**

---

## スライド9: 最小限の構造 - 立ち位置を定義する3要素

**Brainbaseの思想：立ち位置を軸にした3要素**

### 1. 自分（Judgment Core）
- 何を大事にしているか（価値観）
- どう判断するか（判断基準）

### 2. 仕事（Projects + Tasks）
- 今、何に取り組んでいるか
- 今、何をすべきか

### 3. 関係（People）
- 誰と働いているか
- 誰に相談するか・決裁者は誰か

→ **この3つでAIはあなたの「立ち位置」を理解できる**
→ **Markdown + AIだけで始められる**

## スライド10: ハンズオン - あなたの立ち位置を整理しよう

**テンプレート紹介**

```markdown
# 私の立ち位置

## 1. 自分（何を大事にしているか）
- 大切にしている価値観：
- 得意なこと：
- 苦手なこと：

## 2. 仕事（今やっていること）
### プロジェクト
- [プロジェクト名]: [簡単な説明]

### 今週のタスク
- [ ] タスク1

## 3. 関係（誰と働いているか）
- [名前]: [役割・関係性]
  - この人に相談すること：
  - この人が決めること：
```

**所要時間：5-10分**
※配布資料またはQRコードでテンプレート提供

## デモ台本（5ステップ構成）

Brainbaseデモは、**sales-copy-remoteの5ステップ構成**を意識して実施します。
合計約4分（スライド5-7の時間）

### ステップ1: 共感（15秒）

**目的**: 参加者の共感を得る

**台詞例**:
```
皆さん、AIに毎回同じ説明を繰り返していませんか？
「このプロジェクトは...」「この人は...」って。

実は私もそうでした。
毎回説明するのが面倒で、結局使わなくなる。
```

---

### ステップ2: 問題提起（30秒）

**目的**: 問題の本質を示す

**台詞例**:
```
でも、これってAIが賢くなっても解決しないんです。

なぜなら...
AIは「あなたの立ち位置」を知らないから。

あなたが何を大事にしているか、
今何をやっているか、
誰と働いているか。

これを知らない限り、AIは永遠に「初対面の人」なんです。
```

---

### ステップ3: 解決策提示（1分）

**目的**: Brainbaseという解決策を提示

**台詞例**:
```
私は「立ち位置を整理する」ことで、この問題を解決しました。
実際にお見せします。

（スライド5に切り替え）

これがBrainbase。
私の複数事業を統一OSで動かす内部運用システムです。

プロジェクト、タスク、人間関係、過去の決定...
全てがAIに理解された状態になっています。

では、実際に使ってみましょう。
```
