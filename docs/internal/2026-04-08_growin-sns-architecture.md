# グローウィン・パートナーズ SNS自動化システム アーキテクチャ設計

**作成日**: 2026-04-08
**対象案件**: グローウィン・パートナーズ SNS運用・ブランディング支援
**次回MTG**: 2026-04-21 18:00〜（見積もり・フェーズ設計の提案）
**担当**: 佐藤圭吾、神代浩歌（設計・実装）、山本力弥（レビュー・提案）

---

## 1. 目的・背景

グローウィン・パートナーズ（コーポレート系コンサル、社員約100名・年商約20億円）が情報発信の強化を目的にSNS運用の自動化を依頼。

**KPI**: コーポレートブランド認知向上（コンバージョン追求なし）
**指標**: 投稿の平均滞在時間・視聴時間（いいね数ではない）
**ターゲット**: 年商100〜3000億円の中堅企業における管理部門・経営企画責任者層

---

## 2. 自動化 vs 手動の設計原則

> 「コンテンツの判断・品質」は手動、「コンテンツを流す仕組み」は自動化する

| 処理 | 担当 | 理由 |
|------|------|------|
| RSS/記事クローリング | **自動** | 毎日30分以上かかる作業。ゼロにする |
| ニュースキュレーション投稿の草稿生成 | **自動（叩き台）** | 本田さんが1〜2行直すだけの状態まで作る |
| X記事（長文）の草稿生成 | **自動（叩き台）** | CFO EDGE連携・深掘りテーマをAIが構成 |
| 通常投稿（オリジナル）の草稿生成 | **自動（叩き台）** | トレンド・独自見解をAIが短文化 |
| 投稿する記事・テーマの最終選定 | **手動（本田さん）** | 「グローウィンが発信すべきか」の価値判断 |
| 各コンテンツの最終編集 | **手動（本田さん）** | 権威性・信頼の源泉。10〜15分/日の設計 |
| 投稿承認 | **手動（承認フロー）** | DX推進（本田氏）→ マーケ担当の確認 |
| 投稿スケジューリング・実行 | **自動** | 承認後は全自動 |
| YouTube→X自動告知 | **自動** | 動画アップ時に即時連携 |
| パフォーマンス分析・レポート | **自動** | 週次で自動集計 |
| リプライ・コメント返信 | **手動（本田さん）** | 「中の人」感の醸成。週1〜2回15分 |

## 3. プラットフォーム選定

### 3.1 採用スタック

| レイヤー | ツール | 選定理由 |
|------|------|------|
| スケジューラー | **Claude Code Desktop Local Tasks** | Mac Mini常時起動前提。ローカルMCPに直接アクセス可。APIコスト不要 |
| AI処理（コンテンツ生成） | **Claude（Desktop Local Task内）** | RSS収集・記事選定・コメント/X記事/通常投稿の草稿生成を1エージェントで完結 |
| Slack通知 | **Slack MCP（公式）** | Desktop Local TaskからローカルMCP経由で直接呼び出し |
| X投稿・X記事作成 | **xMCP（X公式）** | X Developer Platform公式MCP（xdevplatform/xmcp）。Mac Mini上でローカル動作 |
| Webhook受信 | **Mac Mini上の軽量サーバー** | Slackボタン承認を受け取るエンドポイント |
| インフラ | **Mac Mini + Tailscale + Cloudflare Tunnel** | 詳細は後述 |

> **補足: Claude Codeスケジューリングの3種類**
> - `/loop`: セッション内ポーリング。7日で期限切れ。今回は不使用
> - `/schedule`（Remote Tasks）: Anthropicクラウド実行。ローカルMCP不可・従量課金。今回は不使用
> - **Desktop Local Tasks（採用）**: Mac Mini上で実行。ローカルファイル・xMCP・Slack MCPに直接アクセス可

### 3.2 コンテンツタイプ別の処理担当

| コンテンツタイプ | 生成 | 投稿実行 |
|------|------|------|
| ニュースキュレーション＋コメント | Claude（Remote Trigger） | xMCP |
| X記事（長文・CFO EDGE連携） | Claude（Remote Trigger） | xMCP |
| 通常投稿（オリジナル短文） | Claude（Remote Trigger） | xMCP |
| YouTubeショート告知 | 自動生成（タイトル＋URL） | xMCP |

### 3.3 除外ツールと理由

| ツール | 除外理由 |
|------|------|
| OpenClaw | Desktop Local Tasks + xMCPで代替可能。スタックを分ける理由がない |
| n8n / Dify | 同上 |
| /schedule（Remote Tasks） | ローカルMCPにアクセスできないためMac Mini構成と相性が悪い |
| /loop（CronCreate） | セッション依存。本番不可 |

## 4. システムアーキテクチャ

### 4.1 全体フロー

```
【毎朝 定時起動】
Claude Code Remote Trigger（Anthropicクラウド）
    │
    ▼
RSS/Webクローリング
（CFO・経営管理・AI活用系媒体 + CFO EDGE）
    │
    ▼
Claude が3種類のコンテンツ草稿を生成
  ├─ [A] ニュースキュレーション＋コメント（3〜5本）
  ├─ [B] X記事（長文）草稿（週1〜2本）
  └─ [C] 通常投稿・オリジナル短文（1〜2本）
    │
    ▼
Slack MCP → 本田さんのSlackチャンネルに通知
（草稿一覧 + コンテンツタイプ明示）
    │
    ▼
【本田さんが確認・編集（10〜15分）】
    │
    ├─ 承認ボタン押下
    │       │
    │       ▼
    │   Mac Mini上のWebhookサーバーが受信
    │       │
    │       ├─ 通常投稿 → xMCP → X投稿実行
    │       └─ X記事 → xMCP → X記事として投稿
    │
    └─ 却下 → その日はスキップ

【YouTube Shortsアップ時】
Mac Mini Webhook受信 → Claude が告知文生成 → xMCP → X自動告知
```

### 4.2 インフラ構成

```
Mac Mini（グローウィン事務所設置・24時間起動）
├── Claude Code Desktop App
│   └── Desktop Local Task（毎朝定時）
│       → RSS収集 → AI草稿生成 → Slack MCP通知
├── xMCP サーバー（ローカル動作）
│   └── 承認後のX投稿・X記事投稿を実行
├── Webhook server（軽量サーバー）
│   └── Slackボタン承認受信 → xMCP呼び出し
├── Tailscale
│   └── 神代さん・佐藤さんがSSHでリモート保守
└── Cloudflare Tunnel
    └── Slack WebhookのみWAN公開（固定IP不要）
```

## 5. SNSアカウント設計

| アカウント | 目的 | 運用者 |
|------|------|------|
| ブランド向けアカウント | コーポレートブランド認知・リード獲得 | 本田大和氏（個人顔出し運用） |
| 採用向けアカウント | 中途採用候補者へのリーチ | 別途検討 |

**主要媒体**: X（主軸）＋ YouTubeショート
**優先度低**: TikTok・Instagram（ターゲット層との乖離）

**コンテンツ方針（3タイプ）**:

| タイプ | 内容 | 頻度 |
|------|------|------|
| ニュースキュレーション＋コメント | 業界ニュース＋グローウィン視点 | 毎日3〜5本 |
| X記事（長文） | CFO EDGE連携・深掘りテーマ | 週1〜2本 |
| 通常投稿（オリジナル） | トレンド・独自見解の短文 | 毎日1〜2本 |

- CFO EDGEの記事公開時にXで告知・誘導（自動）
- KPIは滞在時間・視聴時間（いいね数・フォロワー数は副次指標）

## 6. 承認フロー

```
DX推進（本田大和氏）：一次確認・編集
    ↓
マーケ担当：最終承認
    ↓
Slackボタン押下 → Webhook → 自動投稿
```

## 7. フェーズ設計（暫定・4/21MTGで提案）

### Phase 1: 設計・基盤構築（単発）
- アカウント設計・コンセプト確定
- Mac Mini セットアップ（Tailscale・Cloudflare Tunnel）
- xMCP・Slack MCP接続確認
- コンテンツ方針・プロンプト設計

### Phase 2: パイプライン構築（単発〜3ヶ月）
- RSS収集→AI草稿生成→Slack通知フロー実装
- Webhookサーバー実装（承認→X投稿）
- Claude Code Remote Trigger設定
- テスト運用・プロンプト調整

### Phase 3: 伴走運用（月額）
- 週次パフォーマンス分析・レポート
- コンテンツ改善提案・プロンプト最適化
- 障害対応・保守（Tailscale経由）
- YouTube連携追加等の機能拡張
