*01-22 AI最新動向とNECの事業戦略に関する考察*

本会議では、2025年末から2026年初頭にかけてのAI業界の最新動向を包括的にレビューし、NECが今後取るべき戦略的方向性について深い議論を行った。特にアンビエント・エージェント、メモリー戦略、マルチエージェント、AIガバナンス、フィジカルAIという5つの重要トレンドを軸に、NECの強みをどう活かすかを検討した。

*AIプラットフォーム各社の最新動向と戦略分析*

_OpenAIの動向と資金面での懸念_
OpenAIは2024年12月から2025年1月にかけて、GPT-4o-2をリリース（12月11日、Gemini 2.0へのコードレッド対応として長文脈とツール利用を強化）、GPT Image 1.5（12月16日、画像生成が4倍高速化）、GPT-4o-2 Codex（12月18日、エージェント特化型）と矢継ぎ早に新製品を投入した。しかし1月7日のChatGPT Healthリリース後、1月16日に広告テスト導入を発表し、無料版とGoプランに広告を導入する方針を明らかにした。これはサム・アルトマンにとって最終手段とされていた施策であり、資金面での不安が顕在化していることを示唆している。広告はOpenAIが最も弱い領域であり、GoogleやAmazonをバックに持つAnthropic と比較して競争上不利な立場にある。この動きは他のAI企業にも広告モデルを許容するきっかけとなる可能性があり、業界全体への悪影響が懸念される。性能面ではGPT-4o-2 Proが現時点で最強のAIモデルとの評価を維持しているが、使いやすさではClaude Codeに劣るとの意見が出ている。

_Googleの攻勢とAppleとの提携_
Googleは11月18日にGemini 3 Proをリリースし、17日にはGemini 3 Flash（Pro性能を高速・低コストで提供）を投入した。最も重要な展開は12月12日のAppleとの提携発表で、SiriがGeminiを採用することが決定した。これはAppleが独自AI開発を断念し、Googleに依存する決断を下したことを意味し、プラットフォーム戦争における新しい局面の到来を示している。Googleは検索・広告という既存の強みに加え、Appleエコシステムへの組み込みにより、さらなる優位性を確立しつつある。

_Anthropicのエージェント実用化リーダーシップ_
Anthropicはエージェント実用化の最前線に立っており、MCP（Model Context Protocol）をAgentic AI Foundationに寄贈し、特許化せずにオープンな標準規格として推進する姿勢を示した。1月13日にはClaude Code Workを発表し、ネイティブアプリとしてパソコン内のローカルファイル操作、Excel・PowerPoint処理、メール送信、Notion・Slack・Gmailとの連携を実現した。これにより知識労働における ほぼすべてのタスクを自動化できる環境が整った。実際のユーザー事例として、2ヶ月分の溜まったタスクを2時間で完了させたケースが報告され、開発者のキャリア不安を惹起している。さらに驚くべきことに、Claude Code Workアプリケーション自体がほぼ人間のコーディングなしにClaude Codeで開発され、開発期間は1週間程度だったことが明らかになっている。これはシステムインテグレーションやシステム開発というビジネスモデルの終焉を示唆する重大な事例である。1月12日にはClaude 4 Healthcareもリリースされ、医療データ連携にも対応を開始した。

_Metaのベンチマーク不正とオープンソース撤退_
MetaはLLaMA 4でベンチマーク結果の不正があったことを認め、同時にオープンソース戦略からの撤退を表明してクローズドモデルへの転換を発表した。近年のベンチマークは各社が都合の良い指標を切り取って発表する傾向があり、信頼性が低下している。真のベンチマーク指標として、人間の仕事をどれだけ代替できるかという「代替率」が最も重要な指標になるとの見解が示された。

*ダボス会議2026とAI津波発言の衝撃*

ダボス会議2026では「AI津波」という衝撃的な表現が使われ、AIが津波のように労働市場を直撃しており、大半の国や企業は準備できていないとの警告が発せられた。Pixarからはトランプ政権の中国向けAIチップ規制緩和は間違いだとの発言があり、AIが過大評価されすぎているとの指摘もなされた。労働市場へのインパクトはアメリカで特に顕著だが、日本市場でもエンジニア市場を中心に影響が見え始めている。Claude Codeの登場により、多くのエンジニアがコードを書かなくなる世界線に入りつつあることを認識し始め、エンジニアのキャリア設計に大きな影響を与えている。さらにClaude Code Workのようなツールはほとんどのバックオフィスツールや経理ツールを置き換える可能性があり、SaaSスタートアップへの影響も大きい。企業は外部ツールを購入するのではなく、社内で自動化を内製化する方向にシフトすると予測される。ホワイトカラー全般のキャリア設計が今後の重要課題となっている。

*2026年の主要トレンド分析*

_1. アンビエント・エージェント：指示しなくても動くAI_
アンビエント・エージェントとは、指示しなくても裏側で自動的に動作するAIを指す。まだ一般的には普及していない用語だが、2026年に必ず普及する概念である。具体例として、会議中にAIが自動で議事録を取り、アクションを抽出し、会議終了と同時に関係者にタスクを自動振り分けするといった動作が挙げられる。人間は従来通り会議をするだけで、AIに対して明示的な指示を出さなくても、業務プロセスに組み込まれたAIが自律的に作業を実行する。Anthropicの Claude Code Workはファイル操作まで自律的に可能になっており、毎日決まった時間に実行するタスクや、特定のメールが届いたらトリガーされるタスクなど、個人のパソコン内で業務自動化が進む。これを部署単位に拡大すれば、人間の代わりにAIがイベントトリガーや時間トリガーで業務を自動実行する世界が実現する。NECの強みとして、2025年1月に提供開始したKotomi Actがこのトレンドに合致しており、業務ノウハウの自動抽出、組織資産としての蓄積・共有、デジタル業務の自動化実現といった機能を提供できる。

_2. メモリー戦略：記憶が競争優位の源泉_
2025年もメモリー戦略は存在したが、マルチエージェントの流行に隠れて注目されなかった。エージェントがメモリーを管理する方法の優位性が確立されていなかった。課題は長期記憶にあり、AIエージェントは現在コンテキスト内にしか記憶を保持できない。外部記憶として データベースを用意し、必要なタイミングでコンテキストに読み込む仕組みが必要だが、何をどう保存し、どう取得するかというメモリー戦略が未発展だった。2026年はマルチエージェントが成熟し、メモリーに注目が移る年になる。Anthropic内部でも「メモリーが競争優位の源泉」と明言されている。記憶のないAIはただのツールだが、アンビエント・エージェントやAIパートナーとして機能するには、個人の記憶をどれだけ保持できるかが重要になる。ChatGPT Healthは医療データを長期記憶し、Cloud for Healthcareはスマートウォッチデータと連携して記憶化している。NECの強みは「業界特化×長期記憶」にあり、Kotomiの日本語性能と業界特化学習を組み合わせることで差別化できる。金融なら顧客の取引履歴を記憶して最適提案、製造なら設備の稼働履歴を記憶して予知保全、医療なら患者の診療履歴を記憶して継続ケアといった形で、業界知識と長期記憶を組み合わせた他のSI企業には出せないカスタムLLMの提案が可能になる。

_3. マルチエージェント：複数AIの協調_
Gartnerが2026年末までにほとんどのアプリ・サービスの40%がエージェント搭載アプリになると予測している。具体例として、食べログのようなサービスがGUI検索からチャット形式に変わる。ユーザーが「今夜5人で中華、喫煙可能な個室で静かに話せる場所、ここから徒歩圏内、空いているところ」と自然言語で依頼すると、AIがパーソナライズして提案するような形態が標準になる。旅行系やヘルスケア系では既にこうしたサービスがリリースされている。NECの強みはSI力でエージェント連携を実現し、基幹システムとAIエージェントを統合したり、XAIの協調設計を行ったりできる点にある。顧客がエージェント搭載を希望する案件が増加するため、AIエージェントフレームワークを活用して実際の顧客サービスに統合するノウハウが武器になる。

_4. AI規制とガバナンス：NECの信頼が武器_
EU AI Actは違反時に売上高の最大7%という巨額の罰金を科すリスクがあり、ヨーロッパで規制が強化されている。トランプ政権は州のAI法を無効化する署名を出したが、日本では改正個人情報保護法への対応が進んでいる。2025年は未整備だった規制・ガバナンスが、2026年はヨーロッパを中心に急速に強化されている。Grokで発生した未成年の写真を性的に加工する事件など、AIの危険性が銃レベルに達しているため、免許制のような規制・ガバナンス強化が必要との認識が広がっている。NECは2025年12月にAIガバナンスサービス3種（利用状況の監査、リスク評価、規制対応支援）の提供を開始しており、Kotomi導入とセットで提案することでより良い価値提供が可能になる。

_5. フィジカルAI元年：実世界への適用_
2025年がAIエージェント元年だったのに対し、2026年はフィジカルAI元年と呼ばれている。AIがシステム面で発展した後、ロボティクスとして物理的なモーター・サーボの制御に発展し、現実世界での動作に適応するまで時差が発生する。2025年は中国でロボット開発が進み、ロボットオリンピックのようなイベントも開催されたが、まだ遊びの段階だった。2026年は世界への実用化が本格化する。2025年にAIエージェントが市場投入されたが結果は出ていなかったのに対し、2026年はエージェントが実際に結果を出す年になる。同様に、ロボットは2026年に市場投入が本格化する。日本でも工場などにロボットが人間の代替として導入され、従来のロボットアームのような決まった動作だけでなく、人間のように柔軟な思考を持ち人間のインターフェースに合ったロボットが働くようになる。2025年1月には日本のドーナツAIから国産ロボットが発表された。ロボット開発はアメリカ・中国が圧倒的に進んでおり国産ロボットは遅れているが、国防レベルでの懸念から、中国やアメリカ製ロボットを大量導入することは日本の国防上ありえないため、国産ロボティクスの開発が重要課題となっている。日立も取り組んでいるが、NECもロボティクスなどフィジカル領域でAIと絡める方針を明言しており、5G・HAI・産業ロボットの統合提案が有効な戦略となる。

*NECが取るべきアクション*

第一に、Kotomi Actをアンビエント・エージェントの核として推進すべきである。業界特化AIはドメイン知識を強く持つことが差別化のポイントとなるため、業界別に特化したソリューション展開が有効である。

第二に、AIガバナンスを必ずセットで提案することである。世界的にガバナンスへの注目度が高まっており、ガバナンスとセットでAIを提案することで信頼性を担保できる。

第三に、フィジカルAI領域でロボティクスとの連携を強化することである。もともと強いエッジや製造DXと組み合わせることで競争力を発揮できる。

第四に、クライアントゼロの継続が最重要である。AIを顧客に売る前に、まず自社で徹底的に使い倒すことで、社内業務でのKotomi Act全面活用を進め、成功事例を顧客提案に転用し、自分たちが使っているからこそ自信を持って勧められる体制を構築すべきである。

*日本のAI競争力：技術から信頼へのシフト*

シリコンバレーのCEOたちの注目は技術から信頼へシフトしている。サム・アルトマン自身も「安全性が信頼を生む」と語っており、プライバシー・セキュリティ・ガバナンスが競争優位の源泉になりつつある。従来のように技術的先進性やAIリソース（電力・水・GPU）の量だけで勝つ世界ではなくなっている。世論がプライバシー・セキュリティ・ガバナンスに向き始めており、これを競争優位として日本企業も戦える環境が整いつつある。2026年はAIネイティブカンパニーへの変革の年であり、グローバル競争で日本企業が巻き返すことができる最後のチャンスとなる可能性が高い。このチャンスを逃せば、日本企業の競争力回復は極めて困難になると予測される。

*NECの社内環境と課題に関する議論*

会議後半では、NEC社内のAIツール利用環境について率直な意見交換が行われた。NECでは外部のAIサービスを直接使用することが制限されており、NGS（NECジェネレーティブAIサービス）というラッパー経由でGeminiやClaudeを利用する形態となっている。このため、世の中の便利ツールやAIエージェントがほぼ社内で使えない状況にある。他の大手企業では物理的に使えないわけではなく推奨しないレベルや申請制としているケースが多いが、NECは原則禁止という厳格なスタンスを取っている。Claude Code WorkやVS Code拡張機能のClaude Codeなども社内ルール上は使用不可とされている。

一方で、Copilotについては M365が社内デジタルツール群に含まれているため使用可能である。MicrosoftのTeamsやメンバー管理などがMicrosoft系で統一されているため、Microsoft管理下のツールに対するAI導入は比較的ハードルが低い可能性がある。Copilot側でのエージェント活用を進めることが、NEC社内で最も早く実現できる可能性のある選択肢として議論された。ただし、CopilotのAPIが社内で呼び出せるかは確認が必要である。

別の選択肢として、Azure上で社内の適切なサーバーを使用し、シングルサインオンでメンバー管理を行いながら自作のエージェント環境を構築する方法も提案された。実際にAWSのBedrockを使用して開発している事例もある。ただし、長期的には社内資産（ドキュメント等）との データ連携が最も重要であり、Copilotを使ってエージェントを動かせる仕組みを導入することが最も効果が高いとの見解が示された。

NEC社内では、サービス仕様書をもとにしたナレッジ検索の自動化ニーズが高いが、Copilotの利用が進んでいなかったため、独自のNGSにわざわざNotionからデータを登録してRAGとして使用したり、NotebookLMにデータを入れてナレッジ検索したりと、一手間二手間かかっている状況である。データが徐々にBoxに移行されているため、Copilotが直接Boxを参照すればデータ連携がスムーズになり、適用できる業務も増えると考えられる。

2025年11月から12月頃にNEC社内にCAIO（Chief AI Officer）的な役職と、AIを横串で牽引する新部署が設立されたことが話題に上った。参加者の上司がこの組織と若干の接点を持っているため、今回議論された内容（セキュリティリスクを取ってでも最先端を追うべきか、社内での普及活動をどう進めるか等）を上層部と相談する機会を設けることが提案された。下からのアプローチだけでは限界があるため、上層部とのつながりを持つことで、より早く情報を得たり、ポリシー変更の可能性を探ったりすることが重要である。

外部講師からは、他社のSIタスクフォースのトップレベルの人々は「これらのツールを導入しなければ会社として遅れを取りすぎて死ぬ」という認識を持っており、セキュリティリスクはあっても導入を進めなければならないと考えている企業が多いことが共有された。NECの上層部がどのような考えを持っているのか（セキュリティリスク軽減のために世の中から遅れても構わないと考えているのか、リスクを取ってでも世の中についていかなければならないと考えているのか）を確認することが、今後の方向性を決める上で重要であるとの指摘があった。

*他社事例と情報収集の重要性*

会議では、NEC社員が外部の動向や他社の実態を知る機会が限られていることが課題として認識された。社内で限られたツールのみを使用していると、外部で新しく登場するツールについて知る機会がなく、自分からリサーチしない限り情報を得られない。こうした講義や自主的なリサーチを通じて情報をつかんでいく必要性が強調された。話を聞くと使ってみたくなるが社内では使えないというジレンマも指摘された。

次回（2月19日13時予定）では、NECと類似の位置づけにある他社の生成AI利活用状況について情報共有することが合意された。具体的には、セキュリティリスクへの対応方針、社内での普及活動の実態、他社の実際のAI現場の状況など、NEC社内からは見えにくい情報を、公開情報と経験則を組み合わせて提供することが約束された。「他社ができているのにNECができていないのは問題ではないか」という比較材料があれば、社内での議論を前進させやすいとの期待が示された。

*率直な評価：NECとKotomiの位置づけ*

外部講師から、日本企業全体としてOpenAI・Google・Anthropic・中国企業に並ぶAI技術を持つ企業は存在しないとの率直な評価が示された。投資金額が桁で2つ、場合によっては3つ違う状況で、技術的に勝つことは現状不可能である。特にAIはGPU・電力といったリソースをかけた者勝ちの側面があり、日本の国や企業が出せる金額では限界がある。NTTやNECが独自AIを出しているが、独自AIで真っ向勝負するのは無理があるとの見解である。

Kotomiを使う方向性は書いたものの、実際のところオープンソースLLMを最先端で追い、それを日本流にカスタマイズした方が性能は高い可能性がある。オープンソースは半年遅れだが、クローズドモデルを学習・蒸留することで半年後には非常に高い性能が出る傾向がある。現在ではGPT-4o.1レベルのモデルが普通に使える状況になっている。

勝ち筋は、LLM自体での真っ向勝負ではなく、LLMを使った新しいビジネスモデル、日本流のビジネスモデルを作ることにある。日本国内のどのパイを取るかを明確にすることが重要で、例えばNECなら産業系が強いため、産業ロボット系のAI周りを全部取れるような動き方をする。自社の強みとLLMを組み合わせ、日本のどの領域を取るかを固めないと、中国・アメリカの企業が入ってきた時にパワーで負けてしまう。

Kotomiについては、社内でもあまり使われていないのが実態である。唯一の売りは完全クローズドなLLMを実現できる点であり、情報漏洩が理論上絶対にないことである。ただし、これはKotomiでなくても、オープンソースをカスタマイズしてクローズド環境に置けば実現できる。この考え方と業界特化を組み合わせ、ガバナンス・セキュリティに配慮したサービス・SIを提供することが、NECが日本国内の顧客に価値提供できる唯一の道である可能性が高い。
