# 日立製作所・東芝 生成AI利活用状況調査

調査目的: NECコンサルティング向け競合ベンチマーク資料

## 日立製作所

### セキュリティ・利用ポリシー

- **方針: 積極活用（禁止せず推進）**。日立グループは生成AIの社内利用を禁止するのではなく、リスクマネジメントしながら積極活用する方針を明確に打ち出している
- **2023年5月**: 「Generative AIセンター」（CoE組織）を設立。データサイエンティスト・AI研究者に加え、セキュリティ・法務・品質保証・知的財産のスペシャリストを集結し、リスクを管理しながら推進
- **セキュアな社内環境**: Microsoft Azure OpenAI Serviceを活用した「Generative AIアシスタントツール」を整備し、2023年5月末から社内提供開始。クラウド上で機密情報が学習されない仕組みを導入
- **利用ガイドライン**: Generative AIセンターが社内向けに利用ガイドラインを策定、社内相談窓口も設置
- **AI倫理原則**: 2021年策定。外部有識者による「AI倫理アドバイザリーボード」の助言を受け、数百件以上のプロジェクトを評価

### 独自LLM方針

- **方針: 独自LLMは開発しない。汎用LLMをカスタマイズする戦略**
- 「言語モデルをスクラッチから開発するのではなく、パートナーと連携しながら汎用言語モデルを個別カスタマイズする」（日立公式）
- **「LLM工場」構想**: NVIDIAなどと協業し、顧客ごとに業務特化型LLMを構築・運用するサービスを2024年10月より提供開始
  - 「業務特化型LLM構築・運用サービス」
  - 「生成AI業務適用サービス」
- パートナー: AWS, NVIDIA, Google Cloud, Microsoft（Azure OpenAI Service）
- Lumada事業を通じて、顧客のLLM構築支援を事業化

## 東芝

### セキュリティ・利用ポリシー

- **方針: 「非・自前」戦略で外部LLMをセキュアに活用**
- 独自LLMは持たず、OpenAI（Azure OpenAI Service経由）等の外部AIをセキュアな環境で利用
- Microsoft Azure OpenAI Serviceを採用し、**データがクラウド上で再学習に使われない仕組み**を導入。情報セキュリティに配慮
- 「生成AI標準活用基盤」（2024年7月開発発表）: セキュリティ管理と情報保護機能を備えた社内基盤。ソリューション開発の期間短縮と最新生成AI技術活用を両立

### 社内普及・組織

| 項目 | 内容 |
|------|------|
| **推進組織（本体）** | 東芝デジタルソリューションズが生成AI戦略の中核を担う |
| **推進組織（東芝テック）** | 「TEC AI Innovation Hub」（全社タスクフォース）を組織 |
| **CAIO** | 明確なCAIO設置の公表は未確認 |
| **社内トライアル** | 約**400名**が「業務改善トライアル」に参加 |
| **効果** | 参加者1人あたり月平均**5.6時間**の作業削減を達成 |
| **プロンプト共有** | 「社員DX」として業務で活用できるプロンプトを全社員で共有・利用できる仕組みを構築 |
| **AI人材育成** | 「芝大門塾」（東芝ビジネスエキスパート）でAI基礎〜実践まで体系的研修を提供。非技術者（営業等）向けプログラムも用意 |
| **社内コンテスト** | 生成AI活用方法に関する社内コンテストを実施 |

### 活用事例

| 領域 | 内容 | 効果 |
|------|------|------|
| **製造ライン分析** | Azure OpenAIベースのAIマルチエージェントでSMTライン品質データを分析 | わずか**数分**で問題原因究明と改善提案を自動実行。2026年商品化予定 |
| **生成AIマネージドサービス** | 「社員DX」「顧客接点DX」「プロセスDX」「モノづくりDX」の4領域で展開 | 製造業の「現場力向上」「人材不足」解決を支援 |
| **ドキュメント活用** | 「コメンドリ with 生成AI」（生成AI標準活用基盤の第一弾ソリューション） | 社内文書の検索・活用を効率化 |
| **リテールプロモーション** | Transformer + 生成AIでプロモーション最適化AI開発（東芝テック） | 小売業の販促効率向上 |
| **Meister Apps** | 「工程改善アシストパッケージ for SMTライン」（2024年10月提供開始） | データ可視化 + 改善提案 |
| **社員DX** | 定型業務の効率化（月5.6時間/人の短縮） | 400名規模トライアルで効果実証 |

**マルチエージェントMVPの特徴**:
1. 1クリックで使える操作性（キーボード入力不要、生産現場向け）
2. AIの判断根拠を可視化（エージェント間のチャットログを表示。製造業で必須のエビデンス提示）
3. Azure OpenAI + Blazor（Azure App Service）基盤

### 独自LLM方針

- **方針: 独自LLMは開発しない。外部LLMを活用する「非・自前」戦略**
- ChatGPT登場後に独自LLM開発を一時検討したが、「経営再建期に米テック大手と同等のGPU・電力投資は非現実的」と判断し断念
- **NICTとの共同研究**（2025年4月〜）:
  - NICTの**日本語特化型LLM**と検索ツールを活用
  - **RAG（検索拡張生成）の品質向上**に関する共同研究
  - 目的: 社内蓄積文書を活用した**知識継承と業務効率化**
  - 東芝が応用技術（ファインチューニング、RAG精度向上、AIエージェント等）を開発、NICTがLLM基盤を提供
- **AIの進化4段階ビジョン**: 「ツール」→「アシスタント」→「アソシエート（協力者）」→「パートナー」。現在は生成AIにより「アソシエート」段階に到達
