## 1分でわかる AI駆動PM

**AI駆動PM**とは、プロジェクトマネジメント業務にAIエージェントを組み込み、**PM工数を75%削減**する手法です。

```
従来のPM：人が情報収集 → 人が分析 → 人が報告 → 人が意思決定

AI駆動PM：AIが自動収集 → AIが分析 → AIがレポート → 人が意思決定
          ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
          この部分を自動化（PM工数の75%）
```

**キーメッセージ**:
- PMの**面倒な作業**をAIに任せ、**判断と調整**に集中できる
- 既存ツール（Teams/Copilot）で実現可能、新規SaaS導入は不要
- 10人規模PJで**月40時間/PM**の工数削減（年間480時間）

```

  Azure OpenAI Service（独自モデル）

```

### 「自らを不要とせよ」— PM実践の哲学とAI-PMの一致

**brainbaseのPM原則**（出典: Product Management in Practice）:

> **組織化の核心**: 自らを不要とせよ
>
> 自己チェック:
> - 1週間休んでも優先順位付けとデリバリーができるか？
> - 「何をしているのか？なぜ？」に誰でも答えられるか？

**AI-PMによる実現**:

| PM原則 | AI-PMでの実装 |
|--------|-------------|
| **1週間休んでも回る** | manaが日次・週次を自動処理、PMは週末も気にしなくてOK |
| **誰でも状況を把握できる** | 進捗・リスクが自動可視化、ダッシュボードで一目瞭然 |
| **情報に簡単にたどり着ける** | 過去の議事録・タスク・振り返りを自動構造化、検索可能 |

### アウトカム vs アウトプット

**PM実践ガイドの警告**:

| アウトプット（作業） | アウトカム（成果） |
|---------------------|-------------------|
| 機能、リリース | ユーザー価値、ビジネス成果 |
| 測定しやすい | 測定しにくい |
| 短期的 | 長期的 |

**AI-PMの価値**:

```
従来のPM:
  アウトプット作業（報告書作成・進捗集計）に75%の時間
  ↓
  アウトカム思考（ユーザー価値・戦略判断）に25%の時間

AI駆動PM:
  アウトプット作業 → AIが自動化（75%削減）
  ↓
  アウトカム思考に75%の時間を使える
```

**NECへの示唆**: AI-PMは「PMの代替」ではなく、**PMを作業から解放し、本質的な価値創造に集中させる**仕組みです。

## よくある質問（FAQ）

### Q1. セキュリティは大丈夫か？

**A1**: Copilot（M365管理下）は企業データが外部に出ない設計。さらに、Azure OpenAI Serviceを使えばNEC専用環境で運用可能。BAOOは経産省AIガバナンスガイドライン執筆協力者であり、ガバナンス設計も支援します。

### Q2. 既存のPM手法（WF型/アジャイル等）に対応できるか？

**A2**: AI駆動PMは手法に依存しません。ウォーターフォール型でもアジャイルでも、既存のPMプロセスに「AI自動化レイヤー」を載せる形で適用可能です。

### Q3. PMの仕事が奪われるのでは？

**A3**: AI駆動PMは「PMの代替」ではなく「PMの強化」です。ルーティンワークをAIに任せ、PMは**判断・調整・戦略立案**に集中できます。むしろPMの価値が高まります。

### Q4. 小規模PJ（3〜5人）でも効果があるか？

**A4**: 小規模PJでも会議自動化・レポート自動化の効果は出ます。ただし、費用対効果は10人以上の規模で最大化します。統括部では複数の小規模PJをまとめて管理する場合に特に有効です。

### Q5. 導入後、自分たちで運用できるか？

**A5**: Phase 1で技術移転を行い、Phase 2以降は統括部メンバーが自走できる状態を目指します。BAAO/Unsonは「魚を与えるのではなく、釣り方を教える」スタンスです。
