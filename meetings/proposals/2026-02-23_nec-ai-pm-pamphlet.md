## 1分でわかる AI駆動PM

**AI駆動PM**とは、プロジェクトマネジメント業務にAIエージェントを組み込み、**PM工数を75%削減**する手法です。

```
従来のPM：人が情報収集 → 人が分析 → 人が報告 → 人が意思決定

AI駆動PM：AIが自動収集 → AIが分析 → AIがレポート → 人が意思決定
          ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
          この部分を自動化（PM工数の75%）
```

**キーメッセージ**:
- PMの**面倒な作業**をAIに任せ、**判断と調整**に集中できる
- **Azure上でmanaを運用** + M365統合で「使い勝手」と「高度な機能」を両立
- 10人規模PJで**月40時間/PM**の工数削減（年間480時間）

## 技術アーキテクチャ：なぜAzure上でmanaを運用するのか？

### M365標準機能だけでは限界がある

**重要な事実**: Copilot + Power Automateだけでは、manaの高度なAI-PM機能の**40-50%**しか実現できません。

| 機能 | M365標準機能（Copilot Studio + Power Automate） | manaの実装（Azure上） | 差異の影響 |
|------|-----------------------------------------------|---------------------|-----------|
| **定期リマインド** | ✅ 可能（簡易版） | ✅ 12種類の高度なスケジューラー（M1-M12） | M365では単純な定期通知のみ |
| **長期メッセージアーカイブ** | ❌ Teams Message History APIは検索・フィルタが弱い | ✅ Azure Blob Storageで月別/日別アーカイブ | **文脈理解の精度が大幅低下** |
| **高精度AI判断** | △ Copilot Studio（GPT-4ベース、制御性が劣る） | ✅ Azure OpenAI Service（GPT-4o 5.2/5.3） or Bedrock（Claude Opus 4.6） | **AI-PMの質が落ちる** |
| **複雑な状態管理** | △ SharePoint Lists/Dataverse（複雑なクエリが困難） | ✅ Azure Cosmos DB（高速・柔軟なクエリ） | **ワークフロー管理が弱体化** |
| **タスク正本管理** | ❌ SharePoint（Git履歴追跡不可） | ✅ GitHub統合（変更履歴・バージョン管理） | **タスク正本管理が崩壊** |
| **拡張性** | ❌ ローコードの限界（30サービスクラスを管理不可） | ✅ DIコンテナで30以上のサービスクラスを管理 | **運用コストが爆発** |

### Azure + M365統合アーキテクチャ（推奨構成）

```
┌────────────────────────────────────────────────────────────┐
│ Azure側（manaコア：高度なAI-PM機能）                         │
├────────────────────────────────────────────────────────────┤
│ ✓ Azure Container Apps / Functions（mana実行基盤）          │
│ ✓ Azure Blob Storage（Teamsメッセージ長期アーカイブ）        │
│ ✓ Azure OpenAI Service（GPT-4o 5.2/5.3）or Bedrock（Claude Opus 4.6）│
│ ✓ Azure Cosmos DB / SQL Database（状態管理・タスク管理）    │
│ ✓ Azure Private Link + VNet統合（セキュリティ）             │
│ ✓ Azure Monitor（運用監視・アラート）                        │
└────────────────────────────────────────────────────────────┘
                          ↕ Microsoft Graph API連携
┌────────────────────────────────────────────────────────────┐
│ M365側（フロントエンド：社員の使い勝手）                      │
├────────────────────────────────────────────────────────────┤
│ ✓ Teams Webhook（通知配信・ボットインターフェース）           │
│ ✓ Graph API（メッセージ取得、カレンダー連携）                │
│ ✓ SharePoint Lists / Planner（タスク表示・可視化）           │
│ ✓ Copilot Studio（簡易的な対話インターフェース）              │
│ ✓ Power BI（ダッシュボード・レポート可視化）                  │
└────────────────────────────────────────────────────────────┘
```

### NECにとっての価値

| 観点 | 価値 |
|------|------|
| **社員の使い勝手** | M365（Teams/SharePoint）で操作 → 新しいツールを覚える必要なし |
| **AI-PMの品質** | Azure上のmanaコアで高度な機能を実現 → 40-50%の簡易版ではなく**フル機能（100%）** |
| **セキュリティ** | Azure Private Link + VNet統合 → NGS/Box要件を満たす |
| **Microsoft統一** | Azureは Microsoft製 → ガバナンス・サポート体制が一貫 |
| **段階的導入** | Phase 0でM365統合を先行、Phase 1でAzure展開 → リスク最小化 |
| **長期運用** | Azure基盤で拡張性確保 → 全社展開・他部署への横展開が容易 |

### セキュリティ・ガバナンス対応

| 要件 | Azure構成での対応 |
|------|------------------|
| **NGS準拠** | Azure Private Link経由でのみアクセス（インターネット経由不可） |
| **データ保管場所** | Azure Japan East/West リージョン指定（国内データセンター） |
| **認証・認可** | Azure AD統合（NEC社内IDでシングルサインオン） |
| **監査ログ** | Azure Monitor + Log Analytics（全API呼び出しを記録） |
| **データ暗号化** | 保存時・転送時ともに暗号化（TLS 1.3 + AES-256） |
| **バックアップ** | Azure Backup（日次自動バックアップ、30日保持） |

### アウトカム vs アウトプット

**PM実践ガイドの警告**:

| アウトプット（作業） | アウトカム（成果） |
|---------------------|-------------------|
| 機能、リリース | ユーザー価値、ビジネス成果 |
| 測定しやすい | 測定しにくい |
| 短期的 | 長期的 |

**AI-PMの価値**:

```
従来のPM:
  アウトプット作業（報告書作成・進捗集計）に75%の時間
  ↓
  アウトカム思考（ユーザー価値・戦略判断）に25%の時間

AI駆動PM:
  アウトプット作業 → AIが自動化（75%削減）
  ↓
  アウトカム思考に75%の時間を使える
```

**NECへの示唆**: AI-PMは「PMの代替」ではなく、**PMを作業から解放し、本質的な価値創造に集中させる**仕組みです。

## よくある質問（FAQ）

### Q1. セキュリティは大丈夫か？

**A1**: 本提案は**Azure Private Link + VNet統合**によりNGS要件を満たします。データは全てNEC Azure環境（Japan East/Westリージョン）内で処理され、外部インターネットには出ません。Azure OpenAI Serviceを使用することで、LLM処理もNEC専用環境で実行されます。BAOOは経産省AIガバナンスガイドライン執筆協力者であり、セキュリティ・ガバナンス設計も支援します。

### Q2. 既存のPM手法（WF型/アジャイル等）に対応できるか？

**A2**: AI駆動PMは手法に依存しません。ウォーターフォール型でもアジャイルでも、既存のPMプロセスに「AI自動化レイヤー」を載せる形で適用可能です。

### Q3. PMの仕事が奪われるのでは？

**A3**: AI駆動PMは「PMの代替」ではなく「PMの強化」です。ルーティンワークをAIに任せ、PMは**判断・調整・戦略立案**に集中できます。むしろPMの価値が高まります。

### Q4. 小規模PJ（3〜5人）でも効果があるか？

**A4**: 小規模PJでも会議自動化・レポート自動化の効果は出ます。ただし、費用対効果は10人以上の規模で最大化します。統括部では複数の小規模PJをまとめて管理する場合に特に有効です。

### Q5. 導入後、自分たちで運用できるか？

**A5**: Phase 1で技術移転を行い、Phase 2以降は統括部メンバーが自走できる状態を目指します。BAAO/Unsonは「魚を与えるのではなく、釣り方を教える」スタンスです。
