# NEC様向け ナレッジ会話AIソリューション提案書

**提案日**: 2026年2月19日
**提案者**: 一般社団法人ビジネスAI推進機構（BAAO） / 合同会社Unson

---

## エグゼクティブサマリー

NEC様の社内ナレッジ活用における構造的課題（NGS経由のRAG登録負荷、Box移行に伴うデータ分散）を解決するため、**Azure OpenAI / Microsoft Copilot を基盤とした「ナレッジ会話AI」**を提案いたします。

本ソリューションにより、社員は自然言語で社内文書を横断検索・対話的に情報取得でき、**情報検索時間を最大70%削減**、**ナレッジ継承コストの大幅低減**が見込めます。

BAAOのAIガバナンス知見とUnsonのセキュア実装力により、NEC様のセキュリティ要件を満たしながら迅速に導入可能です。

## 1. 課題定義

### 現状の構造的課題

```
現状のナレッジ検索フロー（非効率）

社員 → NGS（NECジェネレーティブAIサービス）
         ↓
     Gemini / Claude（ラッパー経由）
         ↓
     RAG検索 ← Notionデータを手動登録
         ↓
     検索結果を返却

問題点:
  ① Notion → NGSへのRAG登録が手動・高コスト
  ② Box移行でデータソースが分散
  ③ サービス仕様書等の検索に時間がかかる
  ④ ナレッジが属人化し、継承されない
```

### 課題の定量インパクト

| 課題 | 影響範囲 | 推定損失 |
|------|---------|---------|
| RAG手動登録の運用負荷 | ナレッジ管理担当者 | 月間40〜80時間/チーム |
| 情報検索の非効率 | 全社員 | 1人あたり1日30〜60分の検索時間 |
| データソース分散（Notion + Box） | 全部門 | 必要情報の未発見率 推定30%以上 |
| ナレッジ属人化 | 組織全体 | 異動・退職時の知識喪失リスク |

## 2. ソリューション概要：ナレッジ会話AI

### コンセプト

「ナレッジ会話AI」は、社員や顧客との**対話を自然言語でインタラクティブに処理**するAIシステムです。キーワード検索ではなく、「会話するように聞く」ことで必要な情報にたどり着けます。

### 3つの適用領域

| 領域 | 機能 | ユーザー体験 |
|------|------|-------------|
| **社内文書検索AI** | Box / Notion / SharePoint等を横断検索 | 「この仕様書の認証方式はどうなってる？」→ 該当箇所を要約して回答 |
| **顧客対話AI** | FAQ・マニュアルベースの1次対応自動化 | 顧客の問い合わせにリアルタイムで回答、エスカレーション判断も自動 |
| **議事録・ナレッジ継承AI** | 会議内容を自動で構造化・蓄積 | 「先月の〇〇PJの決定事項は？」→ 議事録から抽出して回答 |

### 現状課題との対応

```
ナレッジ会話AI導入後のフロー（効率化）

社員 → ナレッジ会話AI（自然言語入力）
         ↓
     Microsoft Copilot + Azure OpenAI
         ↓
     Box / Notion / SharePoint を直接参照
     （RAG手動登録が不要）
         ↓
     対話形式で回答を返却
     （出典・根拠も提示）
```

**最大の改善点**: Boxを直接参照できるため、NGS経由のRAG手動登録が不要になる

## 3. 期待効果

### 定量効果

| 指標 | 現状 | 導入後 | 改善率 |
|------|------|--------|--------|
| 情報検索時間 | 1人30〜60分/日 | 10〜20分/日 | **60〜70%削減** |
| RAG登録運用工数 | 月間40〜80時間/チーム | ほぼゼロ（自動連携） | **95%以上削減** |
| 問い合わせ1次対応 | 全件人手対応 | 60〜70%自動化 | **工数60%削減** |
| ナレッジ検索ヒット率 | 推定70% | 90%以上 | **+20pt改善** |

### 定性効果

- **データソース一元化**: Box移行完了後も、Notion既存資産とシームレスに横断検索
- **属人化解消**: 対話履歴・検索ログがナレッジとして自動蓄積
- **オンボーディング加速**: 新人が「聞けばわかる」環境により立ち上がり期間を短縮
- **ガバナンス強化**: アクセスログ・利用状況の可視化による情報管理の高度化

### 技術選定の理由

| 技術要素 | 選定 | 理由 |
|---------|------|------|
| LLM基盤 | Azure OpenAI Service | NEC様のAzure環境との親和性、データ国内保持 |
| 対話エンジン | Microsoft Copilot | M365統合、Box Connector対応、権限制御の自動継承 |
| 認証・認可 | Entra ID（旧Azure AD） | NEC様既存ID基盤との統合 |
| ベクトルDB | Azure AI Search | マネージドサービス、Azure OpenAIとのネイティブ連携 |
| 監視・ログ | Azure Monitor | コンプライアンス要件への対応 |

### NGSとの共存方針

既存のNGS環境を置き換えるのではなく、**ナレッジ会話AIを補完的に導入**します。

```
NGS（既存）: Gemini/Claude ベースの汎用生成AI利用
    ↕ 共存
ナレッジ会話AI（新規）: 社内ナレッジ特化の対話型検索
    → 段階的にナレッジ会話AIの適用範囲を拡大
```
