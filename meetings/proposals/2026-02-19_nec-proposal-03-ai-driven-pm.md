## エグゼクティブサマリー

NECが2025年末に新設されたCAIO的役職・AI横串推進組織を最大限に活用し、**全社のAI普及を加速する**ための実行プログラムをご提案します。

本提案の核は**AI駆動PM（AI-Driven Project Management）**——プロジェクトマネジメント業務にAIエージェントを組み込み、進捗管理・リスク検出・会議運営を自動化する手法です。PMという全部門共通の業務プロセスを起点にすることで、「特定部署の取り組み」ではなく「全社の働き方変革」としてAI普及を実現します。

**ソフトバンクは2.5ヶ月で250万エージェント、NTTデータは実践人財7万人を達成しました。**
NECにも同等以上のポテンシャルがあります。足りないのは技術ではなく、**上から火をつける仕組み**です。

## 1. 課題定義：新設AI組織が直面する「3つの壁」

### 壁①：下からのアプローチの限界

現場の有志がAI活用を試みても、部門の壁を越えられない。個人の努力が組織の変革に繋がらない構造的な問題があります。

```
現状のボトルネック：

現場有志 → PoC → 部門内で完結 → 横展開されない → 効果が限定的
                                    ↑
                              ここが壊れている
```

### 壁②：「AI推進」の抽象度が高すぎる

「AIを活用しましょう」では現場は動けない。**何の業務で・どう使い・どう測定するか**が具体化されていないと、組織は動き出せません。

### 壁③：他社との差が開いている

| 企業 | AI普及施策 | 規模 | 成果 |
|------|-----------|------|------|
| **ソフトバンク** | 宮川社長が直接号令。AIエージェント祭を全社実施 | 2万人×2.5ヶ月 | **250万エージェント作成**。90%が肯定的評価 |
| **NTTデータ** | 4段階研修プログラムで全社員AIネイティブ化 | 20万人規模計画 | **実践人財7万人**達成 |
| **日立** | AIアンバサダー37名体制でグローバル展開 | グローバル全社 | トライ&エラー許容カルチャーを確立 |
| **KDDI** | KDDI DX University運営 | 全社員対象 | 全社員プロンプト研修完了 |

**これらの企業に共通するのは、「トップダウン × 具体的な施策 × 測定可能な成果指標」の三位一体です。**

## 2. ソリューション：AI駆動PM手法

### コンセプト：「アンビエントエージェント × PM自動化」

AI駆動PMとは、プロジェクトマネジメントの各プロセスにAIエージェントを**アンビエント（環境的・常時稼働）**に組み込む手法です。

```
従来のPM：
  人が情報収集 → 人が分析 → 人が報告 → 人が意思決定

AI駆動PM：
  AIが自動収集 → AIが分析・異常検出 → AIがレポート生成 → 人が意思決定
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
              アンビエントエージェントが常時稼働
```

### AI駆動PMが実現する5つの自動化

| # | 業務 | 従来 | AI駆動PM |
|---|------|------|----------|
| 1 | **進捗管理** | 週次会議で口頭報告、Excel手動集計 | AIがツール横断で自動集計、異常値を即時アラート |
| 2 | **リスク検出** | PMの経験則に依存 | 過去プロジェクトデータからパターン認識、早期警告 |
| 3 | **会議準備** | PMが手動でアジェンダ作成 | AIが前回議事録・進捗差分から自動生成 |
| 4 | **議事録→タスク** | 手動で議事録作成、タスク振り分け | AIが自動文字起こし→要約→タスク化→担当者振り分け |
| 5 | **ステータスレポート** | PMが毎週手動作成 | AIが自動生成、PMは確認・承認のみ |

### なぜ「PM」を起点にするのか

| 理由 | 説明 |
|------|------|
| **全部門共通** | PM業務はIT部門だけでなく、営業・企画・管理部門にも存在する |
| **効果が測定しやすい** | 工数削減・会議時間短縮・リスク早期検出率は定量化可能 |
| **経営層に刺さる** | 「プロジェクト成功率の向上」は経営指標に直結 |
| **横展開しやすい** | 1部署で成功した型を他部署にそのまま適用可能 |
| **抵抗が少ない** | 「仕事を奪う」ではなく「面倒な作業を減らす」という文脈 |

## 3. 他社事例の深掘り：成功の共通パターン

### ソフトバンク：トップ号令 × 全員参加イベント

- **宮川社長が自ら号令** → 全社2万人が参加せざるを得ない空気を醸成
- **「AIエージェント祭」** → 競争・ゲーミフィケーション要素で自発的参加を促進
- **結果**: 2.5ヶ月で250万エージェント作成、90%が肯定的評価
- **学び**: **トップのコミットメントが最大のブースター**。「やってもいい」ではなく「やるのが当然」の空気づくり

### NTTデータ：4段階研修 × 実践人財の量産

- **4段階の研修プログラム**で全社員20万人のAIネイティブ化を計画
- 「知る → 使う → 作る → 教える」の段階設計
- **結果**: 実践人財7万人を達成
- **学び**: **体系的な育成パスと「教える」レベルまでの引き上げ**がスケールの鍵

### 日立：AIアンバサダー × カルチャー変革

- **AIアンバサダー37名**を各事業部に配置
- トライ&エラーを許容するカルチャーを明示的に推進
- **学び**: **社内の「伝道師」ネットワーク**が横展開のエンジン

### NEC向けの示唆

```
NECに適用すべき成功パターン：

① トップのコミットメント（ソフトバンク型）
   → CAIO組織からの号令で「全社の取り組み」化

② 具体的な施策（NTTデータ型）
   → AI駆動PMという明確な業務適用テーマ

③ 社内伝道師（日立型）
   → パイロット部署のPMをAIアンバサダーとして育成

④ 全社参加の仕掛け（ソフトバンク型）
   → AI駆動PMコンテスト等のイベント設計
```

## 4. 展開ステップ：パイロット → CAIO組織経由 → 全社横展開

### フェーズ全体像

```
Phase 0          Phase 1           Phase 2           Phase 3
設計・準備        パイロット         横展開準備          全社展開
(1ヶ月)          (3ヶ月)           (3ヶ月)            (6ヶ月〜)
─────────────────────────────────────────────────────────────
 現状分析         1部署で実証        成功事例の型化      部門順次展開
 ツール選定        効果測定          研修プログラム化     自走体制構築
 KPI設計          事例化            アンバサダー育成     継続改善サイクル
```

### Phase 0：設計・準備（1ヶ月）

| 項目 | 内容 |
|------|------|
| **現状分析** | 対象部署のPM業務フロー調査、既存ツール棚卸し |
| **AI駆動PM設計** | 自動化対象プロセスの特定、エージェント設計 |
| **KPI設計** | 会議時間削減率、進捗報告工数削減率、リスク早期検出件数 |
| **ツール選定** | 既存環境（Teams/Slack/Jira等）との統合設計 |
| **AIガバナンス** | 経産省AIガバナンスGL準拠のリスク評価・運用ルール策定 |

### Phase 1：パイロット（3ヶ月）

| 項目 | 内容 |
|------|------|
| **対象** | 1部署・2〜3プロジェクト |
| **導入範囲** | 会議自動化（アジェンダ生成→議事録→タスク振り分け）から開始 |
| **段階拡大** | 月次で進捗自動レポート、リスク検出を追加 |
| **効果測定** | 定量（工数削減率）＋ 定性（利用者満足度）で評価 |
| **成果物** | 効果レポート＋経営層向けプレゼン資料（CAIO組織提出用） |

### Phase 2：横展開準備（3ヶ月）

| 項目 | 内容 |
|------|------|
| **成功パターンの型化** | パイロットで得た知見を再現可能なプレイブックに |
| **研修プログラム設計** | 4段階研修（知る→使う→カスタマイズ→教える） |
| **AIアンバサダー育成** | パイロット部署のPM経験者を社内伝道師に |
| **全社展開計画** | CAIO組織と連携した部門別展開ロードマップ |

### Phase 3：全社展開（6ヶ月〜）

| 項目 | 内容 |
|------|------|
| **部門順次展開** | 3〜5部署ずつウェーブ型で展開 |
| **自走体制構築** | NEC社内チームへの技術移管、運用ドキュメント整備 |
| **継続改善** | 月次レビュー、AIモデルの精度改善、新ユースケース開発 |
| **全社イベント** | AI駆動PMコンテスト等のエンゲージメント施策 |

## 5. 費用感と契約形態

### 基本構造：月額顧問 ＋ 研修費

| フェーズ | 内容 | 費用目安（税別） | 期間 |
|---------|------|----------------|------|
| **Phase 0** | 設計・準備 | 月額50〜80万円 | 1ヶ月 |
| **Phase 1** | パイロット導入・伴走 | 月額80〜120万円 | 3ヶ月 |
| **Phase 2** | 横展開準備・研修設計 | 月額80〜120万円 + 研修費 | 3ヶ月 |
| **Phase 3** | 全社展開伴走 | 月額120〜200万円 + 研修費 | 6ヶ月〜 |

### 研修費（別途）

| 研修種別 | 内容 | 費用目安 |
|---------|------|---------|
| **経営層向けワークショップ** | AI駆動PMのビジョン共有（半日） | 50万円/回 |
| **PM向け実践研修** | ハンズオン形式（1日） | 30〜50万円/回 |
| **AIアンバサダー育成** | 集中研修（3日間） | 100万円/回 |
| **全社員向けeラーニング** | コンテンツ制作 + LMS設定 | 200〜300万円（初期） |

### スケールモデル

```
横展開成功 → 対象部署増加 → 研修ニーズ増加 → 契約規模拡大

Phase 1（1部署）  ：月額 80〜120万円
Phase 2（3部署）  ：月額 120〜200万円 + 研修
Phase 3（10部署〜）：月額 200〜350万円 + 研修 + 人員増
```

**横展開に成功するほど、BAAO/Unson側の支援人員も増加 → 継続契約の拡大につながるモデルです。**

## 付録：AI駆動PMの具体イメージ

### Before（従来のPM業務）

```
月曜 9:00  PMがSlack/メールで各担当に進捗確認を依頼
月曜 15:00 集まった情報をExcelに転記
火曜 10:00 進捗会議（1時間）— PMが口頭でヒアリング
火曜 12:00 PMが議事録を手動作成
火曜 14:00 議事録からタスクを手動でJiraに登録
水曜 9:00  PMが週次レポートを手動作成
                                        → PM工数: 約8時間/週
```

### After（AI駆動PM）

```
常時        AIが各ツールから進捗データを自動収集
月曜 8:30   AIが進捗サマリー + 異常値アラートを自動生成
火曜 9:55   AIが前回議事録ベースでアジェンダを自動生成
火曜 10:00  進捗会議（30分）— AIサマリーベースで議論に集中
火曜 10:35  AIが議事録を自動生成 → タスクを自動振り分け
火曜 11:00  PMは確認・承認のみ
水曜 8:30   AIが週次レポートを自動生成 → PMは確認のみ
                                        → PM工数: 約2時間/週
                                        → 工数削減率: 約75%
```
