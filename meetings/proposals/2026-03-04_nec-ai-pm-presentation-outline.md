## タイムテーブル（60分）

### **Part 1: 問題提起 - PMの仕事、このままでいいのか？（8分）**

#### PMの現実

週40時間の内訳:
- 進捗管理: 15時間（各メンバーに確認、Excel更新...）
- 会議準備・運営: 10時間（アジェンダ作成、議事録、タスク振り分け...）
- 報告資料作成: 10時間（週次レポート、月次報告...）
- その他: 5時間

#### 問題点

- PMが「管理作業」に追われて「判断」ができない
- 情報がPMに集中→PMがいないと回らない
- 異常に気づくのが遅い（3週間後に「遅れてます」）
- PMが疲弊→優秀な人ほど辞める

#### 参加者への問いかけ

「皆さんのPMは、本当に"マネジメント"してますか？それとも"作業"に追われてますか？」

### **Part 2: コンセプト - PMの本質的な仕事とは？（10分）**

#### 【2-1】「自らを不要とせよ」— PM実践ガイドからの示唆（5分）

**出典**: Product Management in Practice

**PM実践ガイドの原則**:
「組織化の核心は、自らを不要とせよ」

**自己チェック**:
- あなたが1週間休んでも、チームは回りますか？
- 「今何をしている？なぜ？」に誰でも答えられますか？
- 必要な情報にすぐたどり着けますか？

**多くのPMの現実**:
- ❌ 自分がいないと止まる
- ❌ 自分しか全体が見えない
- ❌ 情報が散らばって探せない

**本来のPMの仕事**:
- ✓ 方向性を決める
- ✓ 優先順位を判断する
- ✓ リスクを見抜く
- ✓ ステークホルダーを調整する

**本来やるべきでない仕事**:
- ❌ 進捗を手動で集める
- ❌ 報告資料を作る
- ❌ 議事録を書く

#### 【2-2】アウトプット vs アウトカム（5分）

**従来のPM**:
```
┌──────────────────────────┐
│ アウトプット作業: 75%     │ ← ここが問題
│ （報告書・進捗集計・会議） │
├──────────────────────────┤
│ アウトカム思考: 25%       │
│ （戦略・判断・価値創造）   │
└──────────────────────────┘
```

**AI駆動PM**:
```
┌──────────────────────────┐
│ アウトプット作業: 25%     │ ← AIが自動化
│ （最終確認のみ）          │
├──────────────────────────┤
│ アウトカム思考: 75%       │ ← 本来の仕事
│ （戦略・判断・価値創造）   │
└──────────────────────────┘
```

**キーメッセージ**: 「AIはループを回す。人間は方向を決める。」

### **Part 4: 実例デモ② - PM業務6つの自動化（10分）**

#### 1. 進捗管理（2分）

**Before**:
PMが毎週15時間かけて:
- 各メンバーにSlackで確認
- Jiraの更新漏れをチェック
- Excelに手動で集計
- 遅延タスクを目視で探す

**After**:
AIが自動で:
- Slack、Jira、GitHub から進捗収集
- 異常値（遅延）を赤く表示
- 依存タスクへの影響を計算
- 「このままだとリリースが3日遅れます」とアラート

**効果**: PMは週3時間、異常値だけ対応（**80%削減**）

#### 2. 会議運営（2分）

**Before**:
PMが週10時間かけて:
- アジェンダ作成（30分）
- 会議中にメモ取り（1.5時間）
- 議事録清書（1時間）
- タスク振り分け（1時間）
- ×週5回繰り返し

**After**:
AIが自動で:
- 前回議事録と進捗差分からアジェンダ生成
- 会議を文字起こし
- 要約を自動生成
- アクションアイテムを抽出→担当者に自動振り分け

**効果**: PMは週1.5時間、確認のみ（**85%削減**）

#### 3. 報告資料作成（2分）

**Before**:
PMが週10時間かけて:
- 進捗データを手動でグラフ化
- 課題・リスクを文章化
- 週次レポートをPowerPointで作成
- 月次報告書を上位職向けに調整

**After**:
AIが自動で:
- 進捗グラフを自動更新
- 課題・リスクを自動抽出
- 週次レポートを自動生成
- 月次報告書を自動生成

**効果**: PMは週1時間、最終確認のみ（**90%削減**）

#### 4-6. その他（簡単に紹介）（4分）

4. **WBS作成**: AIがドラフト生成 → PMがレビュー（**60%削減**）
5. **タスク登録**: 議事録から自動抽出（**70%削減**）
6. **遅延リカバリ**: 過去PJ分析→自動提案（**50%削減**）

### **Part 5: 効果 - 10人規模PJの場合（8分）**

#### 定量効果（数字で見る）

**前提条件**:
- 10人規模PJ（PM 1名 + メンバー 9名、6ヶ月）

**PM工数削減**:
- Before: 週40時間
- After: 週10時間
- 削減: 週30時間（**75%削減**）
- 年間: **1,560時間削減**
- 金額: 1,560h × 5,000円 = **780万円/年**

**会議時間削減（全員）**:
- 週次進捗会議: 1.5h → 0.75h（50%削減）
- 月次報告会: 2h → 1h（50%削減）
- 年間: **510時間削減**（全員合計）
- 金額: 510h × 4,000円 = **204万円/年**

**合計効果**:
- **年間約1,000万円の工数削減効果**（1PJ）
- 10PJ運用の場合: **年間約1億円の効果**

#### 定性効果（目に見えない価値）

**PMの負荷軽減**:
- 戦略的な判断に75%の時間を使える
- 優秀な人が辞めなくなる

**リスク早期検出**:
- 従来: 3週間後に気づく
- AI駆動: 1週間で気づく
- → 2週間早くリカバリ開始できる

**透明性の向上**:
- 誰でもリアルタイムで状況把握
- ステークホルダーの信頼向上

**新人PMの立ち上がり**:
- AIが標準的なWBS・タスクを提案
- 経験の浅いPMでも品質を担保

#### 投資対効果（ROI）

**初年度**:
- コスト: 約1,200万円
- 効果: 約1億2,000万円（10PJ）
- **ROI = 10倍**

**2年目以降**:
- コスト: 運用コストのみ（約400万円/年）
- **ROI = 30倍**

### **Part 6: 導入イメージ - 3ヶ月で効果実証（7分）**

#### パイロットプログラム

**Step 1: 設計・準備（1ヶ月）**

- 業務フロー分析
  - 統括部の実際のPM業務を調査
- パイロットPJ選定
  - 効果が見えやすい10人規模PJ
- KPI設計
  - 何を測るか明確化

**Step 2: パイロット（2ヶ月）**

- Week 1-2: 会議自動化から開始
  - すぐに効果が見える領域
- Week 3-4: 進捗自動レポート
  - PM工数削減を体感
- Week 5-6: リスク検出
  - 安心感が生まれる
- Week 7-8: WBS・報告資料自動化
  - 全体の効果を実感

**伴走体制**: BAAO/Unsonが週1回ミーティング

**Step 3: 効果測定（1ヶ月）**

- PM工数の記録（Before/After）
- 会議時間の記録
- ROI算出
- 上位職への報告サポート

**横展開判断**:
```
パイロット成功
  ↓
上位職の承認
  ↓
3〜5プロジェクトへ横展開
  ↓
全社展開の検討
```

#### 重要な前提

「ツールを入れるだけでは効果30%」
「組織のプロセスを変えると効果200%」

**変わること**:
- ❌ 週次報告会議 → ダッシュボードで常時可視化
- ❌ PMの手動集計 → AIが自動集計
- ❌ 月次レポート作成 → AIが自動生成、PMは承認のみ

**組織の変化**:
- ✓ PMの役割: 「管理者」→「意思決定者」へ
- ✓ 組織文化: 「報告文化」→「透明性文化」へ

### **Part 8: まとめ + Q&A（12分）**

#### まとめ（2分）

**持ち帰ってほしいこと**:

1. **PMの本質的な仕事は「判断」と「調整」**
   - 作業に75%の時間を使っている現状は異常

2. **AI駆動PMは「ツール」ではなく「組織変革」**
   - プロセスを変えることで真の効果が出る

3. **実証済み**
   - BAAO/Unson自身が4ヶ月間実運用中
   - 21名、5PJ、会議時間90%削減

4. **投資対効果が明確**
   - 初年度ROI 10倍
   - 10PJ で年間1億円の効果

5. **3ヶ月で効果実証可能**
   - リスクを抑えてスタートできる

#### Q&A（10分）

**想定質問**:

**Q1**: 「既存のJiraやRedmineと連携できるか？」
**A1**: 可能です。各種PMツールとAPI連携します。

**Q2**: 「セキュリティは大丈夫か？」
**A2**: Azure Private Link + VNet統合でNGS要件を満たします。データは全てNEC Azure環境内で処理。

**Q3**: 「現場メンバーが抵抗しないか？」
**A3**: 「仕事を奪う」ではなく「面倒な作業を減らす」という文脈。むしろ歓迎されます。

**Q4**: 「小規模PJ（3-5人）でも効果あるか？」
**A4**: 小規模でも会議自動化・レポート自動化の効果は出ます。複数PJをまとめて管理する場合に特に有効。

**Q5**: 「導入後、自分たちで運用できるか？」
**A5**: パイロットで技術移転を行い、自走できる状態を目指します。
